# ATS Best Practices Guide (2025 Standards)

## Overview

This document outlines the critical requirements for creating ATS-compliant resumes that pass Applicant Tracking Systems while maintaining human readability.

---

## 1. File Format & Structure

### ✅ Required Format
- **.docx format** - Highest ATS compatibility (not PDF)
- **Single column layout** - Avoid multiple columns
- **Standard section headers** - Use conventional names
- **No headers/footers** - All content in main document body
- **No text boxes** - All text in normal document flow
- **Minimal tables** - Only for simple layouts if necessary
- **No images/graphics** - Text only

### ❌ What to Avoid
- PDF files (lower parsing accuracy)
- Multiple column layouts
- Headers/footers with contact information
- Text boxes or floating elements
- Complex tables with merged cells
- Images, logos, or graphics
- Fancy fonts or decorative elements

---

## 2. Typography & Spacing Standards

### Font Requirements
- **Primary Fonts:** Calibri, Arial, Georgia, Helvetica
- **Body Text:** 11-12pt
- **Section Headers:** 14-16pt
- **Name Header:** 16-18pt

### Spacing Guidelines
- **Line Spacing:** 1.0-1.15
- **Paragraph Spacing:** 10-12pt before/after
- **Margins:** 0.5"-1" on all sides
- **Bullet Points:** Standard bullets (•) only
- **Emphasis:** Bold for headers/company names, minimal italic use

---

## 3. Section Order (Recommended)

```
1. Contact Information
2. Professional Summary
3. Technical Skills / Key Skills
4. Professional Experience
5. Current Projects (if applicable)
6. Education
7. Certifications (optional)
```

---

## 4. Content Optimization

### Keywords
- Match job description keywords naturally
- Include industry-standard terminology
- Use both acronyms and full terms (first mention)

### Action Verbs
Use strong action verbs:
- Architected, Built, Engineered, Developed
- Led, Managed, Coordinated, Facilitated
- Achieved, Delivered, Implemented, Optimized
- Reduced, Increased, Improved, Streamlined

### Metrics & Quantification
Always quantify achievements:
- "Reduced latency by 50%"
- "Architected system for 10K+ users"
- "Achieved 87% model accuracy"
- "Managed migration of 100K+ files"

### Acronyms
First mention format: "Application Programming Interface (API)"
Subsequent mentions: "API"

### Date Format
Consistent format throughout:
- "October 2019 - May 2023" OR
- "Oct 2019 - May 2023"

---

## 5. ATS Parsing Red Flags

### Critical Issues (Will Cause Parsing Failures)
- ❌ Multiple columns
- ❌ Contact info in headers/footers
- ❌ Text boxes
- ❌ Complex tables
- ❌ Images or graphics
- ❌ Unconventional section names
- ❌ Special characters or symbols
- ❌ Fancy fonts (script, decorative, handwriting)

### Warning Issues (May Reduce Accuracy)
- ⚠️ PDF format (use .docx instead)
- ⚠️ Inconsistent date formats
- ⚠️ Abbreviations without context
- ⚠️ Non-standard section names
- ⚠️ Overly creative formatting

## 6. Section-Specific Guidelines

### Contact Information
```
FULL NAME
Professional Title

City, State | Phone | Email
LinkedIn: linkedin.com/in/profile
GitHub: github.com/username | Portfolio: website.com
```

### Professional Summary
- 3-5 sentences
- No brackets or special formatting
- Include years of experience
- Highlight key technologies
- Mention measurable achievements

### Technical Skills
- Organize by category
- List most relevant first
- Include both technologies and methodologies
- Use commas to separate items

### Experience Entries
```
Company Name | Location
Job Title | Date Range

• Action verb + specific achievement + quantifiable result
• Technical implementation details + impact
• Cross-functional collaboration + outcome
```

### Current Projects (For Active Work)
```
Project Name | Role | Status
Live URL | Key Metric

• Technical architecture + scale
• Technologies used + implementation
• Measurable outcomes + impact
```

## 7. ATS Compliance Checklist

Before submitting, verify:

- [ ] .docx format (not PDF)
- [ ] Single column layout
- [ ] Standard fonts (Calibri, Arial, or Georgia)
- [ ] No text boxes or images
- [ ] Standard section headers
- [ ] Contact info in body (not header)
- [ ] Consistent date formats
- [ ] Standard bullets (•)
- [ ] Keywords from job description included
- [ ] All achievements quantified
- [ ] Strong action verbs used
- [ ] 1-2 pages (2 pages for 6+ years experience)
- [ ] No spelling or grammar errors
- [ ] Acronyms explained on first use

---

## 8. Human Readability Checklist

ATS compliance is necessary but not sufficient. Ensure human reviewers can quickly scan:

- [ ] Professional, clean appearance
- [ ] Passes 6-second scan test
- [ ] Clear visual hierarchy
- [ ] Consistent formatting throughout
- [ ] Key achievements stand out
- [ ] Logical information flow
- [ ] Appropriate white space
- [ ] Easy-to-read font sizes

---

## 9. Common ATS Mistakes to Avoid

### Formatting Mistakes
1. Using headers/footers for contact info
2. Fancy formatting (borders, shading, colors)
3. Multiple columns for main content
4. Non-standard bullet points or symbols

### Content Mistakes
1. Missing keywords from job description
2. Vague descriptions without metrics
3. Inconsistent date formats
4. Unexplained acronyms
5. Generic objective statements

### Technical Mistakes
1. Saving as PDF instead of .docx
2. Using incompatible fonts
3. Embedding images or charts
4. Using text boxes for content

❌ Use text boxes
